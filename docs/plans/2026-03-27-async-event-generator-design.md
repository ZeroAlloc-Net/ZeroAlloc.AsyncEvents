# AsyncEvent Source Generator Design

**Date:** 2026-03-27
**Status:** Approved

## Overview

Add a Roslyn source generator to `ZeroAlloc.AsyncEvents` that generates `event AsyncEvent<TArgs>` properties with explicit `add`/`remove` accessors from `AsyncEventHandler<TArgs>` fields annotated with `[AsyncEvent]`.

---

## Section 1 — Attribute & project structure

### Attribute

`src/ZeroAlloc.AsyncEvents/Attributes/AsyncEventAttribute.cs`:

```csharp
[AttributeUsage(AttributeTargets.Class | AttributeTargets.Field, AllowMultiple = false)]
public sealed class AsyncEventAttribute : Attribute
{
    public InvokeMode Mode { get; }
    public AsyncEventAttribute(InvokeMode mode = InvokeMode.Parallel) => Mode = mode;
}
```

### Projects

**`src/ZeroAlloc.AsyncEvents.Generator/`** — new generator project:
- Targets `netstandard2.0`
- `IsRoslynComponent=true`, `EnforceExtendedAnalyzerRules=true`

## Section 2 — Generator logic

### Trigger

A `partial` class qualifies if:
- It has `[AsyncEvent]` on the class itself, OR
- It has at least one `AsyncEventHandler<TArgs>` field with `[AsyncEvent]`

### Mode resolution

Field attribute wins over class attribute:
1. Field has `[AsyncEvent(mode)]` → use field mode
2. Field has no attribute, class has `[AsyncEvent(mode)]` → use class mode
3. Neither → field is skipped

### Field name → event name

Strip leading `_` or `m_`, then PascalCase:
- `_orderPlaced` → `OrderPlaced`
- `m_orderPlaced` → `OrderPlaced`
- `orderPlaced` → `OrderPlaced`

### Generated output

One file per qualifying class (`Namespace_ClassName.AsyncEvents.g.cs`):

```csharp
// <auto-generated/>
#nullable enable

partial class OrderService
{
    public event global::ZeroAlloc.AsyncEvents.AsyncEvent<global::My.OrderPlacedArgs> OrderPlaced
    {
        add    => _orderPlaced.Register(value);
        remove => _orderPlaced.Unregister(value);
    }
}
```

The generator does NOT emit the backing field — users keep full control of field declaration and initialization.

## Section 3 — Testing & packaging

### Snapshot tests (one per scenario)

- Field-level `[AsyncEvent]` with Parallel mode
- Field-level `[AsyncEvent]` with Sequential mode
- Class-level `[AsyncEvent]` — all `AsyncEventHandler<TArgs>` fields pick up the mode
- Field-level overrides class-level
- Mixed: class Parallel + one field Sequential
- Non-partial class → no output, no diagnostic

### Packaging

`ZeroAlloc.AsyncEvents.csproj` bundles the generator:

```xml
<ProjectReference Include="..\ZeroAlloc.AsyncEvents.Generator\ZeroAlloc.AsyncEvents.Generator.csproj"
                  OutputItemType="Analyzer"
                  ReferenceOutputAssembly="false"
                  PrivateAssets="all" />
```

Consumers install one package (`ZeroAlloc.AsyncEvents`) and get attribute + generator automatically.
