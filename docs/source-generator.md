# Source Generator

The `ZeroAlloc.AsyncEvents` package includes a Roslyn source generator that writes `event AsyncEvent<TArgs>` properties with `add`/`remove` accessors automatically. No separate package install is needed — the generator is bundled in the main package.

## Requirements

The declaring class must be `partial`.

## Field-level attribute

Annotate individual `AsyncEventHandler<TArgs>` fields with `[AsyncEvent]`:

```csharp
public partial class OrderService
{
    [AsyncEvent(InvokeMode.Parallel)]
    private AsyncEventHandler<OrderPlacedArgs> _orderPlaced = new(InvokeMode.Parallel);

    [AsyncEvent(InvokeMode.Sequential)]
    private AsyncEventHandler<ItemShippedArgs> _itemShipped = new(InvokeMode.Sequential);
}
```

Generated output (one file per class):

```csharp
// <auto-generated/>
#nullable enable

partial class OrderService
{
    public event AsyncEvent<OrderPlacedArgs> OrderPlaced
    {
        add    => _orderPlaced.Register(value);
        remove => _orderPlaced.Unregister(value);
    }

    public event AsyncEvent<ItemShippedArgs> ItemShipped
    {
        add    => _itemShipped.Register(value);
        remove => _itemShipped.Unregister(value);
    }
}
```

## Class-level attribute

Apply `[AsyncEvent]` to the class to generate events for **all** `AsyncEventHandler<TArgs>` fields:

```csharp
[AsyncEvent(InvokeMode.Parallel)]
public partial class OrderService
{
    private AsyncEventHandler<OrderPlacedArgs> _orderPlaced = new(InvokeMode.Parallel);
    private AsyncEventHandler<ItemShippedArgs> _itemShipped = new(InvokeMode.Parallel);
}
```

## Mode resolution

The `InvokeMode` argument on `[AsyncEvent]` controls which fields are picked up, not the generated accessor code — the accessors are always the same `Register`/`Unregister` delegation. The actual invoke mode is determined at runtime by the `AsyncEventHandler<TArgs>` field initializer.

Field attribute wins over class attribute for inclusion decisions:

| Field has `[AsyncEvent]` | Class has `[AsyncEvent]` | Result |
|---|---|---|
| Yes | — | Field included |
| No | Yes | Field included |
| No | No | Field skipped |

## Field naming

The event name is derived from the field name by stripping a leading `_` or `m_` prefix and uppercasing the first letter:

| Field | Event |
|---|---|
| `_orderPlaced` | `OrderPlaced` |
| `m_orderPlaced` | `OrderPlaced` |
| `orderPlaced` | `OrderPlaced` |

## What the generator does NOT do

- It does not generate the backing field — you declare and initialize it yourself.
- It does not set `InvokeMode` on the field — you control that in the field initializer.
- It does not generate for non-`partial` classes — they are silently skipped.
